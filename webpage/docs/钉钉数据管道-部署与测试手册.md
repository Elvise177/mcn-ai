# 钉钉数据管道 · 部署与测试手册

> 适用：自动化中心二期——从钉钉 AI 表格拉数据（如抖音数据总表）→ 加工 → 写回分表 → 群通知。
> 代码位置：`webpage/lib/automation/dingtalk/`，触发端点 `/api/v1/automation/dingtalk`。
> 角色分工：【钉】= 钉钉管理员操作（客户侧或你自己）；【服】= 服务端操作（开发/你）。

---

## 第一部分：钉钉侧准备（约 20 分钟）

### 1.1【钉】创建企业内部应用

1. 浏览器打开 `open.dingtalk.com`，用**企业管理员**钉钉扫码登录
2. 顶栏「应用开发」→ 左侧「企业内部应用」→ 点「创建应用」
3. 应用名称填 `mcn-ai 自动化`，描述随意，图标可不传 → 创建
4. 进入应用详情 →「凭证与基础信息」页：
   - 记下 **Client ID（即 AppKey）**
   - 记下 **Client Secret（即 AppSecret）**（点眼睛图标显示后复制）

### 1.2【钉】开通权限

1. 应用详情 → 左侧「权限管理」
2. 搜索框输入 `智能表格`（或 `Notable`），把出现的**读、写权限点全部**点「申请权限」→ 状态变为"已开通"
3. 再搜索 `通讯录`，开通「通讯录个人信息读权限」（查 unionId 用）
4. 搜索 `机器人`，开通「机器人消息发送」相关权限（备用）

> 权限开通后即时生效，不需要发布应用版本也能调服务端 API；如遇 `Forbidden`，回这里检查权限点。

### 1.3【钉】拿操作者 unionId

表格 API 要求以某个具体的人的身份操作。用**对目标表格有编辑权限的管理员**：

1. 开放平台顶栏「开发工具」→「API Explorer」
2. 左侧搜索「查询用户详情」（通讯录管理 → 根据 userid 获取用户详情）
   - 不知道 userid？先用「根据手机号查询用户」接口，手机号填管理员的
3. 右侧应用选择刚建的 `mcn-ai 自动化`，点「发起调用」
4. 返回 JSON 里找 `unionid` 字段（一串字母数字），记下来

### 1.4【钉】准备 AI 表格并拿 baseId

1. 手机/电脑钉钉 →「文档」→ 找到抖音数据所在的 **AI 表格**（钉钉数据服务写入的那个文件）
   - 测试阶段也可以新建一个 AI 表格，手工造一张「抖音数据」表填 3 行假数据
2. 用**浏览器**打开该表格（右上角「…」→ 复制链接 → 浏览器打开）
3. 看地址栏 URL，`base/` 或 `notable/` 后面的那串字母数字就是 **baseId**
   - 拿不准就把完整 URL 发给开发，开发来判断
4. 确认表格里有两张工作表（底部标签页）：
   - 源表：钉钉数据服务写抖音数据的表（记下**准确表名**，如 `抖音数据`）
   - 目标表：手工新建一张 `执行明细`，第一行表头至少建这些字段（文本/数字类型）：
     `视频链接`｜`账号名`｜`发布日期`｜`播放量`｜`点赞`｜`评论`｜`GMV`｜`数据更新时间`
5. **确认 1.3 那个管理员对此表格有编辑权限**（表格右上角「分享」看成员列表）

### 1.5【钉】（可选）群机器人，用于同步结果通知

1. 目标钉钉群 → 群设置 → 智能群助手 → 添加机器人 → 「自定义」
2. 安全设置勾「**加签**」，复制 `SEC…` 密钥；完成后复制 Webhook 地址

### ✅ 第一部分产出清单（发给开发/自己收好）

| 项 | 值 |
|---|---|
| AppKey (Client ID) | |
| AppSecret (Client Secret) | |
| 操作者 unionId | |
| 表格 baseId（或完整 URL） | |
| 源表名 | 如 `抖音数据` |
| 目标表名 | 如 `执行明细` |
| 群 Webhook + SEC 密钥（可选） | |

## 第二部分：本地联调冒烟（先于部署，10 分钟）

【服】在 `webpage/` 目录下执行（把尖括号换成真实值）：

```bash
cd webpage
DINGTALK_APP_KEY=<AppKey> \
DINGTALK_APP_SECRET=<AppSecret> \
DINGTALK_OPERATOR_ID=<unionId> \
node scripts/dingtalk-smoke.mjs <baseId> 抖音数据
```

### 预期输出（三步全 ✅）

```
✅ accessToken 获取成功
✅ 工作表 2 张: 抖音数据 / 执行明细
✅ 源表「抖音数据」读到 3 行样例:
   {"视频链接":"https://...","播放量":12345,...}
=== 冒烟通过：把样例字段名发给开发，校准 douyinToDetail 映射 ===
```

### 排障表

| 现象 | 原因 | 处理 |
|---|---|---|
| `❌ 鉴权失败: invalid appkey…` | AppKey/Secret 抄错 | 回 1.1 重新复制（注意别带空格） |
| `❌ 列工作表失败 … Forbidden` | 智能表格权限点没开 | 回 1.2 逐个开通 |
| `❌ 列工作表失败 … 403/无权限` | unionId 那个人看不到这张表 | 回 1.4 第 5 步给他加编辑权限 |
| `❌ 列工作表失败 … 404` | baseId 不对 | 回 1.4 第 3 步重新取，或发完整 URL 给开发 |
| `❌ 读源表失败` | 源表名和实际不一致（多个空格都算） | 用上一步输出的工作表名精确重试 |

### 冒烟通过后：校准字段映射

把冒烟输出的**样例字段名**对照 `webpage/lib/automation/dingtalk/sync.ts` 里的
`douyinToDetail()` 函数——左边是抖音表真实字段名，右边是执行明细表字段名，
不一致就改函数里的键名。改完 `npx tsc --noEmit` 过了再进第三部分。

## 第三部分：线上部署（Vercel，10 分钟）

### 3.1【服】配置环境变量

1. `vercel.com` → 进入项目 → Settings → Environment Variables
2. 逐条添加（Environment 全选 Production + Preview）：

| 变量名 | 值 | 必填 |
|---|---|---|
| `DINGTALK_APP_KEY` | 1.1 的 AppKey | ✅ |
| `DINGTALK_APP_SECRET` | 1.1 的 AppSecret | ✅ |
| `DINGTALK_OPERATOR_ID` | 1.3 的 unionId | ✅ |
| `DINGTALK_BASE_ID` | 1.4 的 baseId | ✅ |
| `DINGTALK_SOURCE_SHEET` | 源表名（默认 `抖音数据`） | 表名不同时填 |
| `DINGTALK_TARGET_SHEET` | 目标表名（默认 `执行明细`） | 表名不同时填 |
| `DINGTALK_UNIQUE_FIELD` | 幂等键字段（默认 `视频链接`） | 字段不同时填 |
| `AUTOMATION_SECRET` | 终端跑 `openssl rand -hex 24` 生成 | ✅ |
| `CRON_SECRET` | 再跑一次 `openssl rand -hex 24` 生成 | ✅（定时用） |
| `DINGTALK_GROUP_WEBHOOK` | 1.5 的 Webhook | 可选 |
| `DINGTALK_GROUP_SECRET` | 1.5 的 SEC 密钥 | 可选 |

### 3.2【服】重新部署使变量生效

Deployments → 最新一条右侧「…」→ Redeploy →（不勾 Use existing Build Cache）→ Redeploy，等状态变 Ready。

### 3.3 定时说明（已随代码配置，无需操作）

`webpage/vercel.json` 已声明 Cron：每天 **UTC 19:00（北京 03:00）** 自动 GET
`/api/v1/automation/dingtalk?run=1`，Vercel 会自动带上 `CRON_SECRET` 鉴权。
要改时间：编辑 vercel.json 的 `schedule`（标准 cron 表达式，UTC 时区），提交推送即生效。
**上线前和客户确认钉钉数据服务每天几点更新抖音数据，Cron 定在其后 1 小时。**

## 第四部分：测试手册（验收逐项打勾）

以下命令里 `$SECRET` 换成 3.1 的 `AUTOMATION_SECRET` 值。

### T1 线上探测（只读，不写任何数据）

```bash
curl -s "https://www.makeupai.top/api/v1/automation/dingtalk" \
  -H "Authorization: Bearer $SECRET" | python3 -m json.tool
```

- [ ] 返回 `"ok": true`
- [ ] `sheets` 数组里能看到源表和目标表
- [ ] `sample` 里有源表前 3 行真实数据
- 返回 401 → SECRET 不对或环境变量没生效（回 3.2 重新部署）
- 返回 500 → 看返回体里的 error（对照第二部分排障表）

### T2 手动执行一次同步

```bash
curl -s -X POST "https://www.makeupai.top/api/v1/automation/dingtalk" \
  -H "Authorization: Bearer $SECRET" | python3 -m json.tool
```

- [ ] 返回形如 `{"ok":true,"result":{"pulled":N,"inserted":N,"updated":0,"skipped":0,"errors":[]}}`
- [ ] 打开钉钉里的目标表「执行明细」：新增了 N 行，字段值与源表对得上
- [ ] 配了群机器人的话：群里收到「抖音数据同步完成 ✅」卡片

### T3 幂等性验证（防重复写入）

紧接着**再执行一次** T2 的命令：

- [ ] 这次 `inserted: 0`，`updated: N`（同样的数据不会重复插行）

### T4 协同保护验证（不碰人工数据）

1. 在钉钉「执行明细」表里**手工加一行**（视频链接填 `人工测试勿删`）
2. 再执行一次 T2

- [ ] 手工加的那行还在、内容没被改
- [ ] 同步照常完成

### T5 增量验证（模拟每天新数据）

1. 在源表「抖音数据」手工加一行新视频数据
2. 执行 T2

- [ ] `inserted: 1`，目标表多了对应一行

### T6 定时任务验证（部署次日早上看）

- [ ] 北京时间 03:00 后：钉钉群收到了同步通知（没配群机器人就看 T6 下一条）
- [ ] Vercel → 项目 → Logs（Functions），筛选 `/api/v1/automation/dingtalk`，能看到 Cron 触发记录且无报错
- [ ] 目标表「数据更新时间」列是当天日期

### T7 告警路径验证（可选但建议）

1. Vercel 环境变量里把 `DINGTALK_APP_SECRET` 临时改错一位 → Redeploy
2. 执行 T2 → 应返回 `"ok": false` 且 error 含"钉钉鉴权失败"
3. **把 Secret 改回来 → Redeploy → 重跑 T2 确认恢复** ✅

### 全部打勾 = 验收通过，可向客户宣布上线

## 附：日常运维

| 事项 | 操作 |
|---|---|
| 看某天同步结果 | Vercel → Logs，按路径筛选；或翻钉钉群通知 |
| 临时手动补跑 | 执行 T2 的 curl（幂等，随时可跑） |
| 暂停定时 | 删掉 vercel.json 里的 crons 段，推送 |
| 换表/换字段 | 改 3.1 对应环境变量 → Redeploy；字段映射变了改 sync.ts 的 douyinToDetail |
| 密钥轮换 | 重新生成 AUTOMATION_SECRET/CRON_SECRET → 更新环境变量 → Redeploy |
| 客户新增一类分表 | 在 sync.ts 加一个 SyncJobConfig（拉一次源表可写多张分表），提交部署 |
