# 分发指南（macOS）

## 现状与原因

当前 dmg 未做 Apple 开发者签名与公证（electron-builder `identity: null`）。
自己电脑上能用；但别人通过**微信/网盘/浏览器**收到文件时，macOS 会打上
`com.apple.quarantine` 隔离标记，Apple Silicon 对未签名应用+隔离标记的组合
直接报**"已损坏，无法打开"**（不是文件真的坏了，右键打开也无效）。

## 临时方案：给测试用户的安装说明（现在就能用）

发 dmg 时附带这段话：

> 1. 打开 dmg，把 mcn-ai 拖进「应用程序」文件夹
> 2. 打开「终端」（启动台搜索 终端），粘贴下面这行，回车，输入开机密码：
>
> ```
> sudo xattr -rd com.apple.quarantine /Applications/mcn-ai.app
> ```
>
> 3. 再双击打开 mcn-ai 即可，只需做这一次

原理：删掉隔离标记，Gatekeeper 不再拦。仅适合发给**信任你的少数测试用户**（Maggie 等）。

## 正式方案：签名 + 公证（对外分发前必做，P1）

1. **注册 Apple Developer Program**（developer.apple.com，个人 688 元/年，审核 1-2 天）
2. **创建证书**：开发者后台 → Certificates → `Developer ID Application`，
   下载后双击装进钥匙串
3. **配置公证凭据**（App 专用密码在 appleid.apple.com 生成）：
   ```bash
   export APPLE_ID="你的AppleID邮箱"
   export APPLE_APP_SPECIFIC_PASSWORD="xxxx-xxxx-xxxx-xxxx"
   export APPLE_TEAM_ID="后台右上角的10位TeamID"
   ```
4. **改 electron-builder.yml**：删掉 `identity: null`，加：
   ```yaml
   mac:
     hardenedRuntime: true
     gatekeeperAssess: false
     notarize: true
   ```
5. `npm run dist` — electron-builder 自动签名+上传公证（首次约 5-15 分钟）。
   验证：`spctl -a -vv release/mac-arm64/mcn-ai.app` 显示 `accepted · Notarized Developer ID`
6. 之后的 dmg 发给任何人都能直接双击打开

## 注意

- 公证需要能连 Apple 服务器（国内网络偶尔慢，挂代理跑 dist）
- 签名后 asar 内容不能再手改，改了必须重新打包
- 自动更新（electron-updater，P1）依赖有效签名，正好一起做
