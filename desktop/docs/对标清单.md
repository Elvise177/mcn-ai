# 对标清单：Claude Desktop + Obsidian

> 用户的原始需求（2026-07 多次明确）："大致按照 Claude Desktop 来开发 + 部分 Obsidian 功能"。
> 本清单是这句需求的执行版：两个基准产品逐项对照，✅=已对齐 ⬜=待补(标优先级) ✖=明确不做(写理由)。
> **每次发包前与基线自查一起过。**

## 对标 Obsidian（知识库模块基准，"部分功能"=以下打钩范围）

### 核心（在范围内）
| 基准行为 | 状态 |
|---|---|
| 本地 md 库、完全兼容 | ✅ |
| 双链解析/点击跳转（含路径式链接） | ✅ |
| 关系图：力导布局/悬停高亮邻居/拖拽/标签 | ✅ |
| 文件树浏览/展开定位 | ✅ |
| 全文搜索（中文） | ✅ |
| 属性（frontmatter）面板化展示 | ✅ |
| 新建/编辑（源码模式）/删除（废纸篓） | ✅ |
| 笔记重命名 | ✅ 2026-07-17 补（自查发现缺失） |
| 反向链接面板（哪些笔记链到这篇） | ⬜ P1 |
| 标签浏览/点击筛选 | ⬜ P1 |
| 模板插入（新建时选模板） | ⬜ P1（脚本卡/复盘卡等业务模板，产品化卖点） |
| 外部改动实时刷新 | ✅ |

### 明确不做（超出"部分功能"）
| 基准行为 | 理由 |
|---|---|
| 插件生态 | 我们的扩展形态是 M7 技能中心 |
| 多标签页/分屏编辑 | 重度编辑场景引导用 Obsidian 打开同一库（兼容即卖点） |
| 白板 Canvas / 幻灯片 | 非目标用户需求 |
| 所见即所得富文本编辑 | MVP 源码模式够用；重编辑走 Obsidian |
| 移动端同步 | 桌面优先，P2 再议 |
